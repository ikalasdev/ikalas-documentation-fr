# FAQ - Questions fréquemment posées

Bienvenue dans la FAQ de l'API Ikalas. Vous trouverez ici les réponses aux questions les plus courantes concernant l'utilisation de notre API.

## Général

### Qu'est-ce que l'API Ikalas ?

L'API Ikalas est une interface unifiée qui vous donne accès à des centaines de modèles d'intelligence artificielle via un seul point d'entrée. Elle simplifie l'intégration de l'IA dans vos applications en proposant une API standardisée.

### Quels types de modèles d'IA sont disponibles ?

Ikalas propose une large gamme de modèles d'IA :
- **Modèles de langage** : GPT-4, Claude, Llama, Mistral, et bien d'autres
- **Modèles d'analyse de documents** : Pour extraire et analyser du contenu
- **Modèles spécialisés** : Pour des tâches spécifiques (traduction, résumé, etc.)

### L'API Ikalas est-elle gratuite ?

L'abonnement à l'API est payant mais certains usages sont gratuits.
Par exemple, si vous utilisez un modèle d'IA gratuit, vous ne serez pas facturé à l'usage mais uniquement pour l'abonnement.
Dans ce cas, là, vous payez uniquement l'abonnement à l'API qui est un prix fixe chaque mois.

Certains usages sont payants, comme l'utilisation de modèles d'IA premium.

## Utilisation de l'API

### Comment faire ma première requête ?

Voici un exemple simple en JavaScript :

```javascript
const axios = require("axios");

const response = await axios.post("https://ikalas.com/api/v1/api-llm-ia", {
  system: "You are a helpful assistant",
  prompt: "Expliquez-moi l'intelligence artificielle en 3 phrases",
  models: "meta-llama/llama-3.3-70b-instruct:free"
}, {
  headers: {
    "apikey": "VOTRE_CLE_API_ICI",
    "content-type": "application/json"
  }
});

console.log(response.data);
```

### Quels sont les paramètres obligatoires ?

Les paramètres obligatoires varient selon l'endpoint, mais généralement :
- `apikey` : Votre clé d'API (dans les headers)
- `prompt` : Votre question ou demande
- `models` : Le modèle d'IA à utiliser

### Comment choisir le bon modèle ?

Le choix du modèle dépend de votre cas d'usage :
- **Modèles gratuits** : Pour les tests et projets personnels
- **Modèles premium** : Pour la production et les performances optimales
- **Modèles spécialisés** : Pour des tâches spécifiques

### Quels sont les formats de réponse supportés ?

L'API Ikalas supporte plusieurs formats :
- `default` : Réponse texte standard
- `json` : Réponse structurée en JSON
- `markdown` : Réponse formatée en Markdown

## Limites et quotas

### Quelles sont les limites de l'API ?

Les limites dépendent de votre plan :
- **Plan gratuit** : Limite de requêtes par minute/jour
- **Plans payants** : Limites plus élevées selon le niveau

### Comment vérifier mon quota restant ?

Vous pouvez vérifier votre quota via le dashboard Ikalas ou en consultant les headers de réponse de l'API.

### Que se passe-t-il si je dépasse mes limites ?

Si vous dépassez vos limites :
- Les requêtes peuvent être rejetées temporairement
- Vous recevrez un message d'erreur 429 (Too Many Requests)
- Attendez quelques minutes avant de réessayer

## Gestion des erreurs

### Quels sont les codes d'erreur courants ?

| Code | Signification | Solution |
|------|---------------|----------|
| 400 | Requête malformée | Vérifiez les paramètres |
| 401 | Clé API invalide | Vérifiez votre clé d'API |
| 403 | Accès refusé | Vérifiez vos permissions |
| 429 | Trop de requêtes | Attendez et réessayez |
| 500 | Erreur serveur | Contactez le support |

### Comment déboguer mes requêtes ?

1. **Vérifiez les logs** : Consultez les détails de l'erreur
2. **Testez avec curl** : Utilisez des outils de test simples
3. **Vérifiez la documentation** : Consultez les exemples
4. **Contactez le support** : Si le problème persiste

### Comment intégrer l'API dans mon application web ?

```javascript
// Exemple d'intégration côté client
async function callIkalasAPI(prompt) {
  try {
    const response = await fetch('/api/ikalas', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ prompt })
    });
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('Erreur:', error);
  }
}
```
